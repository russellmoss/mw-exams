# Methodology: From Corpus to Study System

This document traces the complete research pipeline behind the MW Practical Exam Study System -- from raw past papers to a deployed web application that generates exam-realistic questions, evaluates candidate answers, and provides examiner-calibrated feedback. Every claim is backed by a file reference.

---

## 1. Corpus Assembly

The system begins with a single authoritative document: the complete text of every MW practical exam paper from 2011 to 2025 (14 years, excluding 2020 due to COVID cancellation).

**Source**: `source/MW_Practical_Papers_Compilation.md` (2,585 lines). Contains verbatim question text, wine slot numbers, and examiner-intent annotation sections for each question across all three papers per year.

**Extraction**: A deterministic Python parser (`scripts/parse_source.py`, 224 lines) uses 6 regex patterns to extract the source into three structured JSON files:

| Output | Contents | Count |
|--------|----------|-------|
| `data/exams.json` | Questions with verbatim text, wine slots, mark allocations | 14 years, 42 papers, 153 questions |
| `data/wines.json` | Flat wine catalog with ID, year, paper, slot, full text | 504 wines |
| `data/annotations.json` | Expert reasoning notes per question | 153 records (32 filled, 121 empty) |

The parser does not infer or transform wine names. The `full_text` field preserves wine identity exactly as printed on the exam paper. Wine IDs follow the format `YYYY_pN_wM` (e.g., `2024_p1_w3`).

## 2. Wine Research

Every one of the 504 wines in the corpus was individually researched.

**Output**: `data/wine_research/` -- 504 Markdown files (100% coverage).

Each file contains:
- YAML frontmatter with 20+ fields (producer, vintage, country, region, sub-region, ABV, classification, style category, oak signature, RS level, structural profile)
- Prose sections: quick tasting profile, technical specs, vintage character, producer style, exam rationale
- Named sources (1-4 per file): producer websites, Wine Enthusiast, Decanter, Tim Atkin MW, JancisRobinson.com, CellarTracker, wine-searcher, regional wine board sites

Research was conducted using the Tavily MCP web research tool (`wine-researcher` agent defined in `.claude/agents/wine-researcher.md`). Some files note "Source needed" for specific data points -- these gaps are flagged, not fabricated.

## 3. Expert Annotation

32 of 153 questions (20.9%) have hand-written expert annotations capturing the reasoning a candidate should apply before and during tasting.

**Source**: `data/annotations.json` -- 32 annotations with `is_filled: true`, concentrated in exam years 2015 (10), 2023 (3), 2024 (10), and 2025 (9).

The annotation style is conversational and deductive: start with paper context, name top candidate varieties/regions, then narrow by elimination. Example (2015 P2 Q3):

> "This question is looking at red wines from same country and two regions where 7 & 8 are from one region and 9 and 10 are from another... Upon tasting the wines, its clear that they are old world in nature as they dont have purity of fruit, they are a little rustic and sanguine. A couple wines are high acid, high tannin and relatively high alcohol which suggests Italy and Barolo."

For the remaining 121 unfilled questions, 83 proposed annotations were generated by the `annotation-proposer` agent (`.claude/agents/annotation-proposer.md`), deliberately mimicking the expert's reasoning style. These are stored in `outputs/proposed_annotations/` with `reviewed: false` status -- none have been merged into the canonical annotations store without user review.

## 4. Question-by-Question Analysis (Decision Matrices)

Every historical question was analyzed twice: first from the stem alone (Phase 5A), then through the lens of the master decision trees (Phase 5B).

### Phase 5A: Stem-Only Matrices

**Output**: `outputs/decision_matrices/` -- 112 files.

Each matrix follows a fixed 9-section analytical pipeline:
1. **Question (verbatim)** -- exact IMW wording
2. **Paper context** -- what Paper 1/2/3 tells you before reading the question
3. **Stem signals** -- specific phrases that constrain the answer space
4. **Universe** -- full set of plausible wines given the constraints
5. **Candidate varieties** -- ranked using three confidence tiers: STRONG SIGNAL, PLAUSIBLE, CURVEBALL
6. **Candidate origins per wine** -- region-level predictions per slot
7. **Rule-outs** -- specific reasoning for eliminating alternatives
8. **IMW's likely intent** -- what the examiner is testing
9. **Reality check** -- post-prediction comparison against actual wines, with a "Lesson" takeaway

The three-tier confidence system (STRONG SIGNAL / PLAUSIBLE / CURVEBALL) was used instead of percentages because the 10-year corpus is too small for reliable probability distributions.

### Phase 5B: Tree-Aware Re-Analysis

**Output**: `outputs/decision_matrices_v2/` -- 112 files + 1 README.

V2 matrices add three structural elements absent from v1:
- **Pre-taste matrix**: structured code-block format with paper, family tag, stem summary, mark map, and constraint classification
- **In-taste matrix**: per-wine templates with region lock, classification tier, quality-in-classification language, maturity/potential assessment
- **Anti-collapse rules**: explicit directives like "Do not infer wine N from wine N-1" and "Conservative-calibration rule: when between two tiers, choose the lower one"

V2 frontmatter links each matrix to the relevant master tree via `family_tree_pack_section` file path references, creating a traceable chain from question to tree node.

## 5. Taxonomy & Pattern Extraction

### 5a. Question Taxonomy

An 8-family classification system was designed to categorize every MW practical question by its structural logic.

**Schema**: `outputs/heuristics/question_taxonomy.md` (352 lines) -- defines families F1-F8 with 4-6 subcategories each:

| Family | Name | Count (112 Qs) | Share |
|--------|------|---------------:|------:|
| F1 | Same Variety Comparative | 25 | 22% |
| F2 | Same Origin Comparative | 24 | 21% |
| F3 | Blend / Composition Logic | 6 | 5% |
| F4 | Mixed Identification Breadth | 33 | 29% |
| F5 | Method / Production Dominant | 12 | 11% |
| F6 | Style Mechanism Comparative | 4 | 4% |
| F7 | Hierarchy / Quality Calibration | 8 | 7% |
| F8 | Examiner Curveball / Boundary | 0 | 0% (secondary tag only) |

Source: `outputs/heuristics/question_taxonomy_index.md` (lines 12-18).

**Tags**: `outputs/taxonomy_tags/` -- 112 files, one per question. Each contains YAML frontmatter with family, subcategory, secondary tags, plus body sections: classification reasoning, stem signals, structural features, why this family, rejected alternatives.

### 5b. Examiner Pattern Extraction

**Output**: `outputs/heuristics/examiner_patterns.md` (439 lines) -- 30 numbered heuristics in 4 categories, extracted from the full 112-question corpus.

Key statistics discovered:
- "Same single grape variety" questions: 24 (21% of corpus)
- P1 always includes Chardonnay: 10 of 10 years
- P3 Q1 is sparkling in last 4 years (2021-2024): 100%
- Riesling in P1: 8 of 10 years
- Tokaji in P3: 5 of 10 years
- Average questions per paper: 3.7
- Commercial/market sub-question present: 26% of questions

### 5c. Examiner Report Synthesis

**Output**: `outputs/heuristics/examiner_report_synthesis.md` (197 lines) -- synthesized from 8 practical examiner reports and 5 chief examiner reports across 2017-2025.

Derived 7 "Cardinal Rules" governing how the IMW marks practical papers:
1. Reasoning > Identification (wrong answer with sound reasoning earns marks)
2. Quality must be contextualized (official classifications + price anchors required)
3. No shoehorning (don't force notes to fit a predetermined identity)
4. Answer the question as asked
5. Maturity has 4 required elements (current age, readiness, improvement window, decline horizon)
6. Commercial must be specific and global (channel, geography, price, competitive set)
7. Structural evidence is the foundation

Mark allocation trend discovered: identification dropping from 46% (2022) to 39% (2023-2024), while quality/maturity rising from 22% to 37%.

## 6. Decision Tree Synthesis

Three master decision trees were synthesized from the 112 individual question matrices, plus three family tree packs providing per-family routing detail.

### Master Trees

| Tree | File | Questions analyzed | Lines |
|------|------|--------------------|------:|
| P1 Whites | `outputs/master_trees/p1_whites_tree.md` | 37 | 108 |
| P2 Reds | `outputs/master_trees/p2_reds_tree.md` | 37 | 121 |
| P3 Special | `outputs/master_trees/p3_special_tree.md` | 38 | 140 |

Each tree has two layers:
- **Layer A (Pre-tasting)**: Routes the candidate based on stem signals alone -- question family, wine count, shared constraints, mark allocation. Branches terminate in variety/region predictions with confidence tiers.
- **Layer B (In-glass)**: Sensory overlay that confirms or eliminates Layer A predictions based on what the candidate tastes.

P3 uniquely includes **Layer A.5 (Visual Triage)**: a pre-smell step using visual appearance (bubbles, color family, viscosity) to collapse the P3 universe to a production category before nosing.

Accuracy target: **variety + region** (e.g., "Barossa Shiraz", "Burgundy Chardonnay"). Producer, vintage, and vineyard identification are bonus, not the target.

### Family Tree Packs

| Pack | File | Families covered | Lines |
|------|------|-----------------|------:|
| P1 | `outputs/master_trees/p1_family_tree_pack.md` | F1, F2, F3, F4, F5, F7 | 562 |
| P2 | `outputs/master_trees/p2_family_tree_pack.md` | F1, F2, F3, F4, F7 | 513 |
| P3 | `outputs/master_trees/p3_family_tree_pack.md` | F1, F2, F4, F5, F6, F7 | 594 |

Each family section uses a consistent 10-element structure: corpus count, pre-taste decision path, variety candidates, origin candidates, mark-distribution pattern, curveball risk, in-taste confirmation path, anti-collapse rules, known weak spots, and example questions.

## 7. Backtesting & Iteration

The decision trees were validated using Leave-One-Year-Out (LOYO) cross-validation: train on 9 years, predict the held-out year, repeat for all 10 folds.

### Initial LOYO Results

**Source**: `outputs/backtest_reports/loyo_report.md`

| Metric | Score | Target | Status |
|--------|------:|-------:|--------|
| Top-1 variety | 51.3% | naive+20pp (36.9%) | PASS |
| Top-3 variety | 70.7% | 70.0% | PASS |
| Candidate-set hit | 82.5% | 85.0% | BELOW TARGET |
| MRR variety | 62.3% | -- | -- |
| MRR region | 34.0% | -- | -- |

Naive baseline (always predict most common variety per paper): 16.9%. Trees achieved +34.4pp delta.

Per-paper: P2 Reds strongest (60.8% top-1), P1 Whites weakest (44.4% top-1). Hardest years: 2023 (40.8%), 2025 (42.0%). Easiest: 2016 (70.8%).

Scoring infrastructure: deterministic Python scorer with appellation lookup and synonym normalization, per-wine granular scoring across 5 LOYO score files (`data/loyo_scores_*.json`), full prediction records in `data/loyo_predictions.json` (7,867 lines).

### Audit & Iteration

**Audit**: `outputs/backtest_reports/loyo_audit_2015_2018_2024_2025.md` identified 5 scoring logic defects and classified misses into four buckets: scoring artifact, ranking miss, open-set miss, category/style miss.

**Iteration 2**: `outputs/backtest_reports/iteration_report.md` -- 20 holdout questions (2023-2024): top-1 variety 52.7%, top-3 86.2%, candidate-set 100%.

### Post-Fix Results

**Source**: `outputs/backtest_reports/loyo_postfix_audit.md`

After unifying the scoring pipeline and updating master trees:

| Metric | Before | After | Delta |
|--------|-------:|------:|------:|
| Top-1 variety | 51.3% | **72.8%** | +21.5pp |
| Top-3 variety | 70.7% | **89.2%** | +18.5pp |
| Candidate-set | 82.5% | **95.6%** | +13.1pp |

Specific tree changes that improved scores: Melon de Bourgogne/Muscadet added as P1 survivor, Rhone-specific same-region blend rule for P2, non-Champagne commercial sparkling leaf for P3, five-mechanism sweet-wine rule, Spain same-country multi-category branch with anti-collapse rule.

### Exam Structure Predictor

A separate sequence-aware structure predictor was backtested on 2022-2025 (`outputs/backtest_reports/exam_predictor_backtest.md`):

| Metric | Score |
|--------|------:|
| Structure mean F1 | 0.499 |
| Variety top-3 hit | 59.5% |
| Country top-3 hit | 81.0% |
| Style top-3 hit | 97.6% |
| Question-role top-3 hit | 92.9% |

Predicted exact question count per paper correctly in every year (2022-2025). Output stored in `data/predicted_2026_exam_profile.json`.

| Appellation varieties | `data/appellation_varieties.json` | 201 appellations | Legal grape variety lookup for stem-constraint validation |

### Three-Layer Constraint Enforcement

**Layer 1 -- Agent instructions** (`.claude/agents/mock-exam-writer.md`, ~300 lines of constraints):
- 10% repeat cap on producer+cuvee across all mocks
- Anti-pattern lock: no question type >60% in same slot position
- Country concentration: max 6 per paper, max 8 absolute ceiling
- Variety ledger mandatory for every question
- Curveball budget: max 1 per question, 2-3 per paper
- Sweet wine mechanism diversity minimums (3 for 4-wine, 4 for 5-wine flights)
- Price ceiling: $400 per bottle absolute maximum
- Method hook ledger for P1 production-method questions

**Layer 2 -- Prompt guardrails** (`study-app/src/lib/prompts/question-generation-prompt.ts`):
- Paper scope constraint (P1=white still, P2=red still, P3=special only)
- Same-origin variety diversity guardrail with blend overlap detection
- Different-country truthfulness constraint
- Latest-question anti-repeat
- P3 wine appearance notes (visual-only descriptions)
- Output sanitization preventing answer leakage in metadata

**Layer 3 -- Server-side validation** (`study-app/src/app/api/get-question/route.ts`, 5 validators):
1. `validatePaperScope` -- regex-based wine-type enforcement (30+ grape patterns per paper)
2. `validateVarietyConsistency` -- appellation-aware variety detection against "same variety" stems
3. `validateOriginDiversity` -- primary-variety overlap detection in same-origin questions
4. `validateCountryDiversity` -- country-count enforcement against "different countries" stems
5. `validateNoveltyAgainstLatest` -- Jaccard similarity check preventing back-to-back similar questions

Up to 5 generation attempts; all validators must pass with zero violations. Failed attempts trigger regeneration with violations logged.

### Mock Exam Output

**Output**: `outputs/mock_exams/` -- 11 full mock exam versions (v1-v11), each with 3 papers, 12 wines per paper, examiner-realistic stems, and model answers. Supporting documents include sourcing guide, country concentration reference, coverage analysis, rotation blueprint, and wine bank report.

## 10. Agent Architecture

The system uses 12 specialized agents (defined in `.claude/agents/`) plus 1 shared rules file:

| Agent | Role in pipeline |
|-------|-----------------|
| `wine-researcher` | Web research per wine (504 files) |
| `annotation-proposer` | Draft expert annotations for unfilled questions (83 files) |
| `question-analyst` | Produce decision matrices per question (112 v1 + 112 v2 files) |
| `taxonomy-tagger` | Classify questions by family/subcategory (112 tag files) |
| `heuristics-extractor` | Cross-corpus pattern extraction (13 heuristic documents) |
| `pattern-synthesizer` | Recurring logic within paper x family buckets |
| `tree-synthesizer` | Synthesize master decision trees from matrices (6 tree files) |
| `tree-backtester` | LOYO backtest and iteration scoring |
| `matrix-writer` | Pre-taste and in-taste study matrices |
| `taxonomy-auditor` | Quality control across the full taxonomy system |
| `mock-exam-writer` | Full 3-paper mock exam generation |
| `mock-answer-writer` | 8-minute model answers in blind-tasting deductive style (112 files) |

All agents share non-negotiable rules (`_shared_rules.md`): citation requirements, no-invention policy, confidence tiers, MW-specific vocabulary, decisive style.

## 11. Deployed Application

The study app (`study-app/`) is a Next.js application deployed on Vercel at `https://study-app-blond-nine.vercel.app`. It provides:

- **Question generation**: LLM-generated questions with three-layer constraint enforcement
- **Pre-glass reasoning**: Stem analysis with visual appearance cues for Paper 3
- **Tasting simulation**: Per-wine sensory note generation
- **Answer writing**: Timed answer input with voice dictation
- **Evaluation**: Examiner-calibrated feedback using the Seven Cardinal Rules, model answer comparison, pass/fail estimation with per-sub-question marks
- **Study diagrams**: Interactive decision trees at `/diagrams/` with zoom, print, and PDF export
- **Feedback loop**: User feedback on generated questions drives pipeline improvements (validated against the corpus before accepting)

The evaluation prompts reference the examiner report synthesis, decision trees, and confidence tiers to ensure feedback is calibrated to actual IMW standards rather than generic AI assessment.
