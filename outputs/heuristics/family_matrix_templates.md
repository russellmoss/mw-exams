# Family Matrix Templates

## Purpose

This document gives the candidate a standard skeleton to use when writing pre-taste and in-taste matrices for each taxonomy family. The matrices in `outputs/decision_matrices/` and `outputs/decision_matrices_v2/` should follow these templates so that:

1. Every question can be triaged into a known matrix shape within seconds.
2. The candidate writes matrices in the same way they will write the exam answer — structured, decisive, classification-aware.
3. The family tree packs in `outputs/master_trees/p{1,2,3}_family_tree_pack.md` plug directly into these matrix shapes.

Each template has two halves:

- `Pre-taste matrix` — drafted from stem only, before any glass evidence.
- `In-taste matrix` — drafted as a per-wine record once the wines are in front of the candidate.

## Universal pre-taste matrix header

Every pre-taste matrix should open with the same five lines, regardless of family:

```
Paper:        P1 / P2 / P3
Family:       F1–F8 (and subcategory if known)
Stem summary: one sentence
Mark map:     where the marks actually are
Constraint:   the binding clue (variety / origin / blend / method / classification / structure)
```

This is non-negotiable. It forces the candidate to register the family and the binding constraint before any candidate guessing.

## Universal in-taste matrix header

```
Wine slot:  W{n}
Snapshot:   one sentence sensory phrase
Style:      still / sparkling / sweet / fortified / oxidative
Acid:       low / med / med-high / high
Body:       light / medium / full
Sugar:      dry / off-dry / medium / sweet / very sweet
Alcohol:    estimated band
Oak / lees / oxidation: read in one phrase
```

Then the family-specific blocks below.

## F1 Same Variety Comparative Set

### F1 pre-taste matrix

```
Variety lock: the bound grape
Comparative axis: climate / region / classification / winemaking / maturity
Sub-form:      F1a same-country / F1b cross-country / F1c same-region / F1d cross-style (P3)

Candidate map (by slot):
  slot 1: most plausible origin × confidence tier
  slot 2: most plausible origin × confidence tier
  slot 3: ...
  slot 4: ...

Anti-collapse rule: never put the same origin in two slots unless the stem allows.
Single rule-out worth naming: the one classic origin that the stem excludes.
```

### F1 in-taste matrix (per wine)

```
Confirm the bound grape: one signature aroma/structure
Position on the comparative axis: cool / warm / classic / new / young / mature
Origin narrowed: country → region → sub-region only as far as evidence supports
Method comment: one phrase on winemaking lever
Quality + maturity: in classification language
Branch-B alternative: one fallback candidate
```

## F2 Same Origin Comparative Set

### F2 pre-taste matrix

```
Origin lock: country / region as bound by stem
Internal divergence axis: variety / sub-region / classification / age / sweetness
Sub-form:    F2a / F2b / F2c / F2d

Candidate map (by slot):
  slot 1: variety or sub-style × confidence tier
  slot 2: ...
  slot 3: ...

Anti-collapse rule: do not predict the same variety twice within a same-country flight unless the stem allows.
```

### F2 in-taste matrix (per wine)

```
Confirm the bound origin: one regional signature
Place on the divergence axis: which variety / sub-region / tier / age
Sub-region narrowed
Classification or tier
Quality in regional context
Branch-B alternative
```

## F3 Blend / Composition Logic Set

### F3 pre-taste matrix

```
Blend family lock: Bordeaux / Rhône / Iberian / Tuscan / Cape / Loire / Vinho Verde / etc.
Sub-form:          F3a / F3b / F3c / F3d
Composition prior: dominant variety + supporting varieties

Candidate map (by slot):
  slot 1: origin within blend family + likely composition split
  slot 2: ...

Anti-collapse rule: keep at least one slot for the non-canonical origin (e.g. Stellenbosch on a Bordeaux-variety flight).
```

### F3 in-taste matrix (per wine)

```
Family signature
Composition reading (dominant + supporting + estimated proportion)
Origin narrowed via family + climate
Quality / classification in blend-family language
Reason this blend works: what each grape contributes
For F3d only: explicit blend-vs-varietal contrast
```

## F4 Mixed Identification Breadth Set

### F4 pre-taste matrix

```
Pure breadth (F4a) / partly linked (F4b) / hidden theme (F4c)
Theme (if any): used as filter, not narrower

Slot ballots (independent, no cross-inference):
  slot 1: best-guess grape / origin × confidence
  slot 2: ...
  slot 3: ...
  slot 4: ...
  slot 5: ...

Hard rule, written at the top of every F4 matrix:
"Do not infer wine N from wine N−1."
```

### F4 in-taste matrix (per wine)

```
Sensory snapshot
Best-guess grape
Best-guess origin
Branch-B alternative
Method comment if marks demand
Quality tier
Explicit reset note before the next slot.
```

## F5 Method / Production Dominant Set

### F5 pre-taste matrix

```
Method family lock: sparkling / sweet / fortified / oxidative / unusual elevage
Sub-form:           F5a sparkling / F5b oxidative / F5c fortification / F5d sweetness / F5e human-vs-natural / F5f style-commercial

Production-family slots:
  slot 1: production family + most plausible region
  slot 2: ...
  slot 3: ...
  slot 4: ...
  slot 5: ...

Anti-collapse rule: do not put the same production family in two slots if the stem says "different methods of production."
```

### F5 in-taste matrix (per wine)

```
Method family in one phrase
Key production lever named (e.g. tirage, biological ageing, fortification timing, appassimento)
Style consequence in palate language tied to method
Origin narrowed only as far as evidence supports
Quality / commercial position framed in method + classification terms
Branch-B alternative method
```

## F6 Style Mechanism Comparative Set

### F6 pre-taste matrix

```
Structural axis: sweetness × alcohol / tannin × ripeness / maturity / commercial position
Sub-form:        F6a sweetness / F6b alcohol / F6c tannin / F6d maturity / F6e quality / F6f climate

Axis plot (slots positioned on axis before tasting):
  slot 1: corner of axis
  slot 2: ...

Anti-collapse rule: spread predictions across at least three points on the axis.
```

### F6 in-taste matrix (per wine)

```
Estimated RS band
Estimated alcohol band
Position on structural axis
Most plausible production mechanism for that position
Origin and variety as confirmation
Quality and commercial position
```

## F7 Hierarchy / Quality Calibration Set

### F7 pre-taste matrix

```
Hierarchy type:  classification ladder / producer hierarchy / age-maturity ladder
Sub-form:        F7a / F7b / F7c / F7d
Region or system lock

Ladder slots:
  slot 1: tier × confidence
  slot 2: ...
  slot 3: ...

Anti-collapse rule: do not predict the same tier for two slots unless the stem allows.
Conservative-calibration rule: when between two tiers, choose the lower tier.
```

### F7 in-taste matrix (per wine)

```
Category and family
Most plausible classification tier (with one alternative)
Quality justified in classification language (length, complexity, integration, concentration)
Commercial position in channel terms
Producer / cuvée guess optional and last
Maturity / potential
```

## F8 Examiner Curveball / Boundary Set

Reserved for cases where the stem itself signals the boundary-testing nature. F8 matrices follow the F4 template with one extra block at the top:

```
Boundary signal noted (one sentence on what the stem is breaking)
Default-frame rejected (one sentence on the lazy answer to avoid)
```

## Pre-taste discipline (applies to all families)

- Write the universal header *before* looking at the wines.
- Lock the family and binding constraint before guessing candidates.
- Spend at least one minute per question on rule-outs, not just candidates.
- Where the marks favour style / commercial / classification language, write the matrix in that language from the start.

## In-taste discipline (applies to all families)

- One wine at a time. No mental contagion.
- The branch-B alternative is mandatory unless the candidate is highly confident — exam markers reward acknowledging uncertainty.
- Classification language belongs in the matrix, not just adjectives. "Likely Grand Cru rather than premier cru, on the basis of length and concentration" beats "very good quality".
- Commercial position is a separate sentence, not an afterthought.

## How these templates relate to the existing decision matrices

- The existing `outputs/decision_matrices/` files are predominantly stem-only (Phase 5A) and follow a free-form layout. They should be re-formattable into these family-aware shapes.
- The Phase 5B re-analysis (`outputs/decision_matrices_v2/`) should explicitly adopt these templates so that the family tree packs and the per-question matrices share one vocabulary.
- When a new mock exam is written, the matrix author should select the template above by family tag before writing prose.

## Maintenance

- Templates evolve only when a pattern recurs across at least two corpus questions and the existing template demonstrably fails to capture it.
- Single-question patches do not justify template changes.
- If a new family is added to the taxonomy, add a matching pre-taste + in-taste template here.
