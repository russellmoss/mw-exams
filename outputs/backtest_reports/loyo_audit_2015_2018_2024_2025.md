# LOYO Audit: 2015, 2018, 2024, 2025

Purpose: audit the scoring logic and classify misses in the weakest held-out years without proposing question-specific patches.

## 1. Scoring logic audit

### Finding A: `top1_region_subregion` is not comparable across fold families

- In `scripts/aggregate_loyo_scores.py`, the 2017-2018 normalizer maps `top1_region_subregion` from `top1_region_hit` when present rather than from a true subregion field.
- Evidence: `normalize_2017_2018` uses `scores["top1_region_hit"] / n` for `top1_region_subregion`.
- Impact: the reported subregion metric for 2018 is effectively a country/region-top1 metric, not a subregion metric.

### Finding B: 2024-2025 country and subregion metrics are internally inconsistent

- In `scripts/aggregate_loyo_scores.py`, the 2024-2025 normalizer computes:
  - `top1_region_country` from per-wine `region_top1`
  - `top1_region_subregion` from per-wine `subregion_hit`
- In the raw 2024-2025 scored outputs, many questions have `predicted_regions` equal to `Unknown` or a partial list, while `subregion_hit` is still `true` on most wines.
- Examples:
  - `2024_p1_q1`: `predicted_regions = Unknown|Australia`, `top1_country = 0.0`, `top1_subregion = 0.8333`
  - `2024_p3_q3`: `predicted_regions = Unknown`, `top1_country = 0.0`, `top1_subregion = 1.0`
  - `2025_p1_q3`: `predicted_regions = Unknown`, `top1_country = 0.0`, `top1_subregion = 1.0`
- Impact: country and subregion accuracy are not measuring one coherent origin-prediction object in these folds.

### Finding C: 2024-2025 `top3_region` is inherited, not recomputed from per-wine rows

- In `scripts/aggregate_loyo_scores.py`, `normalize_2024_2025` sets `top3_region = round(item["scores"]["top3_region"] * n)`.
- Impact: this metric is being reverse-engineered from a stored question-level proportion rather than recomputed from per-wine ranks like the other fold families. It is therefore less auditable.

### Finding D: subregion scoring in the deterministic scorer is string-containment on free text

- In `scripts/run_loyo.py`, `top1_region_subregion` is incremented when the actual subregion string appears inside `origins_per_slot[slot]["raw"]`.
- This is not tied to `region_ranking`.
- Impact: a wine can fail country ranking but still get subregion credit if the raw slot note mentions the place name.

### Finding E: several audited misses were already marked parser-sensitive

- `scripts/run_loyo.py` marks these as `PARTIAL_PARSE_FILES`:
  - `2024_p1_q3`
  - `2024_p2_q2`
  - `2025_p1_q3`
  - `2025_p3_q3`
- Impact: these questions should not drive tree edits until scoring and parse representation are cleaned up.

## 2. High-level miss taxonomy

This audit uses four buckets:

- `Scoring artifact`: the miss is partly caused by inconsistent metric plumbing.
- `Ranking miss`: the right family survived, but the top-1 order is wrong.
- `Open-set / mixed-bag miss`: the tree candidate list is too narrow or too weakly calibrated for a grab-bag structure.
- `Category/style miss`: the tree tries to force a varietal answer where the question is really about style, production method, or category.

## 3. Year-by-year taxonomy

### 2015

Primary pattern: real tree weakness, but mostly ranking weakness rather than candidate-set failure.

#### 2015 ranking misses

- `2015_p2_q2` different countries / different varieties:
  - Tempranillo, Shiraz, and Gamay all survived, but only Tempranillo ranked top-1.
  - Safe interpretation: ranking calibration problem, not a missing-family problem.
- `2015_p2_q3` same-country two-regions:
  - Nebbiolo finished rank 2 behind Sangiovese in Piedmont slots.
  - Safe interpretation: Italy regional ordering problem inside a correct candidate set.
- `2015_p2_q4` same-region different varieties:
  - Merlot lost top-1 to Cabernet Sauvignon in Napa.
  - Safe interpretation: benchmark Bordeaux-blend ordering issue.
- `2015_p3_q3` sweet wine survey:
  - Riesling in Canada was right by variety but wrong by country rank.
  - Furmint and Trebbiano were present but not top-1.
  - Safe interpretation: sweet-wine ranking across countries is weak.

#### 2015 open-set / mixed-bag misses

- `2015_p1_q3` six different varieties:
  - Candidate-set hit is 100%, but top-1 variety is only 16.7%.
  - The model knew the six possibilities but could not assign them to slots reliably.
  - Safe interpretation: this structure needs better slot-level ranking logic, not narrower trees.

#### 2015 category/style misses

- `2015_p3_q1` and `2015_p3_q2` are style-level questions scored through varietal resolution.
- Candidate-set hit is 100% in both, but top-1 variety is only 25%.
- Safe interpretation: this is a tree architecture issue on P3. The system recognizes the category families but not the varietal mapping priority.

### 2018

Primary pattern: mixed-bag and style/category failures, with one scoring oddity in recorded subregion reporting.

#### 2018 ranking misses

- `2018_p1_q1` same-variety Chardonnay flight:
  - Variety is perfect, but region order degrades across France, USA, New Zealand.
  - Safe interpretation: region ranking weakness within a correct variety lock.
- `2018_p2_q1` benchmark red survey:
  - Pinot Noir, Syrah, Merlot, Tempranillo all survive somewhere in the list, but only one slot is top-1.
  - Safe interpretation: benchmark-red ordering is weak when several classic families co-occur.

#### 2018 open-set / mixed-bag misses

- `2018_p1_q2` six-wine grab-bag:
  - top-1 variety `16.7%`, top-3 variety `33.3%`, candidate-set `83.3%`
  - Tempranillo misses entirely from candidate set on one slot.
  - Garganega, Chenin Blanc, Torrontes are ranked too low.
  - Safe interpretation: open-set white mixed-bag logic is too shallow.

#### 2018 category/style misses

- `2018_p3_q1` and `2018_p3_q2` are again the hard P3 category questions.
  - `2018_p3_q2` is especially bad: top-1 variety `0.0%`, candidate-set `66.7%`, despite region being perfect.
  - Safe interpretation: the tree can read the country/region frame but is not representing the right grape/style mappings cleanly.

### 2024

Primary pattern: heavy scoring distortion on origin metrics plus genuine P3 and blend/appellation weakness.

#### 2024 scoring-artifact-heavy questions

- `2024_p1_q1`, `2024_p2_q2`, `2024_p3_q2`, `2024_p3_q3`:
  - `predicted_regions` is `Unknown` or nearly empty.
  - `top1_country` is near zero while `top1_subregion` is high.
  - Safe interpretation: do not tune the tree off country misses here until the origin scorer is made coherent.

#### 2024 ranking misses

- `2024_p1_q1`:
  - top-3 variety is `100%`, candidate-set is `100%`, but top-1 variety is only `33.3%`.
  - Safe interpretation: P1 comparative white ranking is decent, but top-1 ordering is too rigid.
- `2024_p1_q2`:
  - same pattern on aromatic whites: all three wines are in top-3, but only Viognier is top-1.
  - Safe interpretation: aromatic-white ordering problem, not a missing-family problem.
- `2024_p2_q3`:
  - Pinot Noir is perfectly identified by variety, but country ordering across five countries is weak.
  - Safe interpretation: multi-country same-variety region ranking needs calibration.

#### 2024 blend/appellation misses

- `2024_p1_q3`:
  - top-1 variety `0.0%`, candidate-set `33.3%`
  - actual wines are a Manseng blend, a Bordeaux red blend, and a Rhone white blend
  - predicted varieties collapse to `Semillon|Grenache`
  - Safe interpretation: blend-led same-country questions are under-modeled.
- `2024_p2_q2`:
  - actual set includes Nebbiolo, Sangiovese, a Bordeaux-style blend, and Tempranillo/Garnacha
  - predicted varieties are only `Nebbiolo|Sangiovese|Tempranillo`
  - Safe interpretation: “classic Europe” logic is too single-variety-centric for mixed red sets.

#### 2024 category/style misses

- `2024_p3_q1`:
  - total collapse: all metrics `0.0`
  - Safe interpretation: sparkling / non-Champagne style-family logic is not robust enough.
- `2024_p3_q4`:
  - top-1 variety `20%`, candidate-set `20%`
  - actuals span Vidal, Austrian sweet blend, Vin Santo blend, Muscat, Chenin Blanc
  - Safe interpretation: residual-sugar / sweet-style questions still need a category-first tree.

### 2025

Primary pattern: severe under-specification on origin output plus several real P1/P2/P3 logic misses.

#### 2025 scoring-artifact-heavy questions

- `2025_p1_q2`, `2025_p1_q3`, `2025_p2_q1`, `2025_p2_q2`, `2025_p3_q2`:
  - `predicted_regions` is `Unknown` or partly empty.
  - subregion hit is often `100%` while top-1 country is `0.0%`.
  - Safe interpretation: treat country metrics cautiously until the scorer is fixed.

#### 2025 blend/appellation misses

- `2025_p1_q1`:
  - one wine is Bordeaux white (`Cabernet Sauvignon/Merlot` in the scored file, likely parser or label issue), one is Sauvignon Blanc from Margaret River.
  - Candidate-set only `50%`.
  - Safe interpretation: this question is not reliable enough to use as a tree-tuning anchor without inspecting the underlying actual labels.
- `2025_p1_q3`:
  - French same-country whites: Melon de Bourgogne, Riesling, Chenin Blanc
  - predicted only `Sauvignon Blanc|Riesling|Chenin Blanc`
  - Melon de Bourgogne is missing entirely.
  - Safe interpretation: French same-country white branch needs a Muscadet/Melon survival rule.
- `2025_p2_q2`:
  - all four wines are Rhone-family, but the prediction is Rioja-family only.
  - candidate-set `0.0%`
  - Safe interpretation: this is a real branch-selection failure, not a ranking issue.

#### 2025 open-set / mixed-bag misses

- `2025_p2_q3`:
  - actuals are Zweigelt, Barbera, Gamay, Xinomavro, plus a Portuguese reference point
  - prediction collapses to a Port-grape family list
  - candidate-set remains `100%` only because one family overlaps somewhere, but top-1 and top-3 variety are both `20%`
  - Safe interpretation: all-European indigenous red grab-bag needs a broader “regional signatures” branch.

#### 2025 category/style misses

- `2025_p3_q2`:
  - Cava blend branch identifies one slot, then fails on Garnacha and Palomino within the same Spanish-style comparative set.
  - Safe interpretation: P3 category branch needs multi-style-within-country handling.
- `2025_p3_q3`:
  - catastrophic single-lock failure: predicted variety is only `Riesling` for a six-wine mixed set spanning Provence rose, Alsace Pinot Gris, Tokaji sweet, Madeira, and Port.
  - top-1 variety `0.0%`, candidate-set `0.0%`
  - This file is also in `PARTIAL_PARSE_FILES`, so both parsing and tree logic may be involved.
  - Safe interpretation: never allow a P3 mixed-category branch to collapse to a single grape lock.

## 4. Safe improvement directions

These are the changes that are least likely to overfit:

- Unify scoring before further tree edits.
  - Recompute all metrics from per-wine rows in one scorer.
  - Make country and subregion derive from the same origin object.
  - Remove `Unknown` as a ranked country output when subregions are explicitly present.

- Split P3 logic into category-first branches.
  - sparkling / traditional method
  - oxidative vs non-oxidative
  - fortified
  - sweet wine mechanism
  - mixed-category “do not single-lock variety” branch

- Improve mixed-bag calibration rather than adding niche leaves.
  - The repeated failure mode is not “missing one exact grape”.
  - It is “too few families survive” or “one family is over-promoted too early”.

- Add blend/appellation survival rules in P1/P2.
  - French same-country white sets need Melon de Bourgogne/Muscadet to survive.
  - Classic Europe mixed reds need a branch that can retain blends plus single-variety signatures together.

- Add an anti-collapse rule.
  - If a question is mixed-bag, category-led, or same-country-different-varieties, the tree should not output a single-grape lock unless the stem explicitly says same variety.

## 5. What not to do

- Do not tune to 2024/2025 country metrics until the origin scorer is fixed.
- Do not patch single questions that are already in `PARTIAL_PARSE_FILES`.
- Do not use 2015/2018/2024/2025 style-category questions to add narrow varietal leaves; that would mostly optimize the scorer, not the exam strategy.

## 6. Recommended next step

Before any new tree revision, fix the LOYO metric pipeline so all fold families emit one consistent per-wine schema. Then rerun the aggregate report and re-audit only the questions that remain true misses after scorer cleanup.
